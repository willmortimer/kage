Hardware-Backed Key Management

Kage acts as a secure bridge between your developer workflows (age/SOPS) and hardware security modules (Secure Enclave, TPM). It eliminates key sprawl and protects secrets at rest.

🔑

Single Source

One master key (K_org) stored only in 1Password. No local copies.

⚙️

Deterministic

Per-environment keys derived via HKDF. Zero configuration drift.

🛡️

Hardware Wrapped

Keys on disk are encrypted by Secure Enclave or TPM.

👻

Ephemeral

Identities exist in memory only during decryption commands.

Authentication Policy Balance

Kage balances security friction against protection needs. "Strong" policies bind keys to specific biometric identities or PCR states, preventing unauthorized usage even with shell access.

Visualizing the defensive capabilities. Kage excels at physical defense and reducing key sprawl but does not protect against full kernel compromise.

Directory Structure

Strict separation of configuration and sensitive data.

  • ~/.local/share/kage/
  • ├── config.toml // Config
  • ├── wrapped/ // Data
  • │ └── {org_id}/

Core Commands

The primary interface for initialization and SOPS integration.

Initialization
kage init --org-id myorg --env prod --1p-vault DevOps
SOPS Integration
export AGE_IDENTITIES_COMMAND="kage age-identities --env=prod"
Recovery
kage rotate-device-key --env prod
